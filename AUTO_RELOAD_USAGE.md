# 自动重载功能使用说明

## 📋 功能介绍

自动重载管理器会监控主程序的运行时间，在达到设定时间（默认12小时）后自动重启软件，确保程序始终保持最佳运行状态。

## 🎯 使用场景

- ✅ **长时间运行**：需要连续多天使用软件
- ✅ **防止内存泄漏**：定期重启释放系统资源
- ✅ **保持稳定性**：避免长时间运行导致的性能下降
- ✅ **自动化管理**：无需手动干预，全自动运行

## 🚀 启动方式

### 方式1：双击批处理文件（推荐）

直接双击运行项目根目录下的：
- **`启动-自动重载.bat`**

### 方式2：命令行启动

```bash
python auto_reload.py
```

## 🔧 配置参数

### 修改重载时间

编辑 `auto_reload.py` 文件，找到以下行：

```python
RELOAD_HOURS = 12  # 12小时后重载
```

修改为你需要的时间（单位：小时）：

```python
RELOAD_HOURS = 24  # 24小时后重载
RELOAD_HOURS = 8   # 8小时后重载
RELOAD_HOURS = 6   # 6小时后重载
```

### 修改主程序路径

如果需要监控其他程序，修改：

```python
manager = AutoReloadManager(
    main_script="app.py",  # 改为你的程序文件名
    reload_hours=12
)
```

## 💡 使用提示

### 正常流程

1. **启动管理器**
   ```
   双击 启动-自动重载.bat
   ```

2. **输入验证码**
   - 主程序启动后会要求输入每日验证码
   - 输入正确的6位数字验证码

3. **正常使用**
   - 像平常一样使用软件
   - 管理器在后台自动计时

4. **自动重启**
   - 12小时后会自动关闭并重启
   - 重启后需要重新输入验证码

### 手动控制

- **停止管理器**：按 `Ctrl+C`
- **查看状态**：控制台会实时显示运行时间
- **意外退出**：如果主程序崩溃，会询问是否重启

## ⚠️ 注意事项

### 重要提醒

1. **验证码需要重新输入**
   - 每次自动重启后都需要重新输入验证码
   - 请确保你知道当天的验证码

2. **保存工作进度**
   - 自动重启前不会保存未完成的任务
   - 建议在接近重启时间时完成当前任务

3. **不要同时启动多个实例**
   - 只运行一个自动重载管理器
   - 避免多个主程序同时运行

4. **系统时间准确性**
   - 确保系统时间准确
   - 时间不正确会影响重载计时

### 常见问题

**Q: 如何知道什么时候会重启？**

A: 控制台会实时显示倒计时，例如：
```
⏱️  已运行: 11:45:30 | 距离下次重载: 00:14:30
```

**Q: 重启时会丢失数据吗？**

A: 重启是强制性的，不会等待当前任务完成。建议在重启前完成正在进行的操作。

**Q: 可以暂停计时吗？**

A: 目前不支持暂停功能。如需暂停，请按 `Ctrl+C` 停止管理器。

**Q: 重启失败怎么办？**

A: 如果重启失败，管理器会显示错误信息。检查：
- Python环境是否正常
- app.py文件是否存在
- 是否有权限执行程序

**Q: 可以在服务器上使用吗？**

A: 可以，但建议使用nohup或systemd等工具让其在后台运行：
```bash
nohup python auto_reload.py > reload.log 2>&1 &
```

## 🔍 故障排查

### 问题1：管理器无法启动

**症状**：运行bat文件后立即退出

**解决**：
1. 检查Python是否正确安装
2. 确认app.py文件存在
3. 查看错误提示信息

### 问题2：主程序无法启动

**症状**：管理器运行但主程序窗口不出现

**解决**：
1. 单独运行 `python app.py` 测试
2. 检查是否有其他实例在运行
3. 查看防火墙/杀毒软件是否阻止

### 问题3：计时不准确

**症状**：实际运行时间与显示不符

**解决**：
1. 检查系统时间是否正确
2. 确认电脑没有进入睡眠模式
3. 重启管理器重新计时

### 问题4：自动重启失败

**症状**：到达时间后没有重启

**解决**：
1. 查看控制台是否有错误信息
2. 检查进程是否正常终止
3. 手动停止后重新启动管理器
